Lighthouse SOC
What

A role-based SOC triage platform with separate workflows for:

Reporter
Analyst / Responder
Admin / SOC Lead
Why

This makes the project feel like a real security operations product instead of a script with extra screens.

How

Build one application, with:

shared backend
shared database
role-based routes
role-based permissions
different screens per user group
1. Core roles
Reporter

Purpose: submit alerts or suspicious activity.

Typical user:

employee
helpdesk agent
internal user reporting phishing or suspicious behavior

Can:

submit new alert
view their own submitted alerts
add extra evidence to their own reports

Cannot:

view analyst queue
see other incidents
access metrics or admin settings
Analyst / Responder

Purpose: triage and work incidents.

Typical user:

junior SOC analyst
responder
security operations staff

Can:

view alert queue
open incidents
enrich context
add notes
assign severity/confidence
change status
escalate
close incidents
view playbook suggestions

Cannot:

manage users
change system rules unless explicitly allowed
access admin-only reporting controls
Admin / SOC Lead

Purpose: oversight, analytics, governance.

Typical user:

team lead
SOC manager
security admin

Can:

view all incidents
view analytics and trends
manage users
review escalation rates
review false positives
manage playbooks and rule settings
export reports

May also:

impersonate analyst visibility for oversight
reassign incidents

9. Best build path
Phase 1 — CLI-first engine

Build:

alert intake
enrichment
severity/confidence scoring
SQLite logging
role-based menus

This proves the logic.

Phase 2 — lightweight UI

Use Streamlit.

Create pages:

Reporter
Analyst Queue
Incident Detail
Admin Dashboard

This gives you screenshots and polish fast.

Phase 3 — app-like structure

Optionally move to Flask later if you want:

proper routes
login/session handling
RBAC decorators

But you do not need that on day one.

10. Recommended tech choice

For your current goal, I’d do:

MVP stack
Python
SQLite
JSON
Streamlit

Why:

fast to build
visual
easy role-based pages
easy charts
easy screenshots for portfolio

You can still describe it as:

a Python-based SOC triage platform with role-based workflows

That is fine.
