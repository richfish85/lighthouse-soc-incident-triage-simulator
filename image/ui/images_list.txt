RECOMMENDED FORMAT AND SOURCING RULES
- Prefer SVG for all logos, illustrations, maps, and icons.
- Avoid raster screenshots as production assets; keep the reference mockups only as design guidance.
- Avoid copying the mockups asset-for-asset; recreate the lighthouse illustration and logo system as original repo-owned assets.
- Keep the palette aligned with the prototype: deep navy/charcoal base, off-white text, restrained gold accent, contained severity colors.
- Use one icon family across all roles so the stack can change later without redoing the asset language.
